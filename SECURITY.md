# 安全政策

本專案是**群益期貨 CapitalAPI 的 AI 可讀規格庫**（文件＋預建知識圖譜），不是可執行的服務或應用程式，傳統意義上的「安全漏洞」（如 RCE、注入攻擊）在此不適用。以下說明回報範疇與管道。

## 回報範疇

### 一般規格錯誤 → 公開 Issue 或 PR

規格內容有誤、遺漏，或與官方文件、範例碼行為不一致，這類問題**不涉敏感資訊**，請直接使用 [規格勘誤模板](.github/ISSUE_TEMPLATE/spec_correction.md) 開 Issue，或依 [CONTRIBUTING.md](CONTRIBUTING.md) 直接送 PR。

### 敏感問題 → 私下回報

以下情況請**不要**開公開 Issue，改寄信到 `<CONTACT_EMAIL>`：

> **維護者請注意**：請將上方 `<CONTACT_EMAIL>` 佔位字串換成實際可用的聯絡信箱，否則此回報管道形同虛設。

- 官方文件抽取（`api_spec/_raw/`）或範例碼中疑似殘留真實帳號、密碼、憑證、金鑰等個資或敏感資訊
- CI/CD、GitHub Actions 設定的供應鏈疑慮（例如懷疑某個已釘選 SHA 的 action 版本遭竄改或投毒）
- 懷疑有 PR／Issue 意圖植入誤導性規格內容，可能導致他人依規格生成的下單程式碼造成金錢損失
- 其他任何你認為不適合公開討論、需要維護者優先私下處理的問題

回報時請盡量附上：問題描述、影響範圍（哪些檔案／哪個函式規格）、發現或重現方式。維護者會盡力（best effort）在收到後回覆，但本專案為非商業性質的社群專案，無法保證回應時效（SLA）。

## 重要免責聲明

- 本專案為**非官方**整理文件，一切以群益期貨官方公告與技術文件為準，如有出入請以官方為準
- 任何依據本規格庫生成的下單、改單、刪單相關程式碼，**務必先在模擬環境（測試帳號）驗證邏輯正確**後，才可切換正式環境操作；下單類操作涉及真實金流與帳務風險
- 本專案維護者對規格內容的準確性、完整性，或因使用本規格庫而產生的任何金錢損失，不負任何責任（詳見 [LICENSE](LICENSE) 的免責條款）
- 著作權與授權相關問題請見 [NOTICE.md](NOTICE.md)；行為規範請見 [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md)

## 支援範圍

本專案目前僅維護對應 CapitalAPI **V2.13.57** 版本的規格內容。若群益期貨發布新版官方文件而規格庫尚未更新，請開 Issue 提出（附官方新版文件連結或版本號），而非視為安全問題。
